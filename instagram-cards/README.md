# 기분 스튜디오 — 인스타그램 카드뉴스 작업 폴더

Claude Design 핸드오프(`반복 연애 패턴 카드뉴스.html`)를 베이스로 한
**다크모드 + 오렌지 액센트** 톤의 1080×1350 카드뉴스 템플릿.

## 폴더 구조

```
instagram-cards/
├── README.md                       ← 이 파일 (사용법)
├── shared.css                      ← 모든 카드 공통 스타일
├── _design-reference/
│   ├── original-template.html      ← 원본 디자인 (7장 다 들어있는 미리보기)
│   └── design-tokens.md            ← 색상·폰트·사이즈 명세
└── cards/                          ← 생성한 카드 1장 = HTML 1개
    └── (사용자가 카피 주면 여기에 생성됨)
```

## 디자인 시스템 한눈에 보기

| 항목 | 값 |
|---|---|
| 캔버스 | **1080 × 1350** (인스타그램 4:5 세로형) |
| 배경 | `#0e0e10` (거의 블랙) |
| 액센트 | `#e85d3a` (오렌지) |
| 폰트 | Pretendard (CDN) |
| 톤 | 차분한 에세이 + 강한 강조어 (오렌지) |

## 사용 가능한 카드 슬롯 6종

원본 템플릿에 들어있는 카드 종류예요. 카피 줄 때 어떤 슬롯으로 만들지
같이 알려주시면 됩니다.

| 슬롯 | 용도 | 핵심 요소 |
|---|---|---|
| **COVER** | 첫 장 (캐러셀 표지) | 큰 타이틀(3줄) + 서브타이틀 + → 화살표 + 도트 배경 |
| **CHECKLIST** | "이런 적 있나요?" 진단 | 헤드라인 + ✓ 체크박스 5개 |
| **TEXT** | 일반 본문 (정의/설명) | 헤드라인 2줄 + 본문 단락 1~2개 (강조어 가능) |
| **LOOP** | 사이클/루프 다이어그램 | 4개 노드 원형 배치 + 점선 화살표 |
| **MOMENT** | 인용/포인트 강조 | 본문 + 큰 인용구(왼쪽 액센트 보더) + 본문 |
| **STEPS** | N단계 실천법 | 헤드라인 + 번호 단계 + 마무리 한 줄 |

> **혼합도 가능해요.** 예: 본문 텍스트 + 마지막에 단계 박스 = TEXT+STEPS

## 카피 전달 형식 (이렇게 주시면 됩니다 🎯)

가장 단순한 형식은 **자연어**예요. 예를 들어:

```
1장 (COVER)
서브: 매일 아침이 무거운 사람을 위한 가이드
타이틀: 번아웃, 혹시 나도?
강조: "나도?"의 ?만 오렌지

2장 (CHECKLIST)
헤드라인: 혹시 / 이런 적 있나요?
체크 5개:
- 일이 끝나도 마음이 안 쉬어진다
- 주말이 와도 무기력하다
- 작은 일에도 짜증이 난다
- 잠을 자도 피로가 안 풀린다
- 좋아했던 것들이 시들해 보인다

3장 (TEXT)
카테고리: PATTERN
헤드라인: 이건 / 게으름이 아니에요
본문: 번아웃은 ...
강조어: "감정의 신호"는 오렌지(.hl), "오래 참아온 결과"는 흰색 굵게(.strong)
```

**필수로 알려주실 항목:**
- 슬롯 종류 (COVER/CHECKLIST/TEXT/LOOP/MOMENT/STEPS)
- 헤드라인 (줄바꿈 위치 표시 — `/` 기호 추천)
- 본문/항목 텍스트
- 강조 단어 (어떤 색으로 강조할지: **오렌지** or **흰색 굵게**)

**자동으로 들어가는 항목:**
- 페이지 번호 (`02 / 07` 같은 푸터)
- `NEXT →` 표시
- 브랜드 핸들 (`@gibun.studioo` — 변경 필요시 알려주세요)

## 작업 흐름

1. 시리즈 시작할 때 → 총 몇 장인지 + 시리즈 카테고리(예: `RELATIONSHIP · 02`) 알려주세요
2. 장마다 카피 전달 → 제가 `cards/01-cover.html`, `cards/02-checklist.html` 식으로 생성
3. 다 만들면 미리보기용 `cards/preview.html`도 자동 갱신
4. 최종적으로 PNG 변환 (Puppeteer/dom-to-image) — 이건 나중에 셋업

## PNG 변환 (나중에)

각 카드는 1080×1350 native로 만들어져있어 그대로 캡처 가능합니다.
필요해지면 `npx puppeteer` 기반 스크립트나 Chrome DevTools의 "Capture full size screenshot"으로 추출.
